# Telegram Global Search Script

Скрипт для **глобального поиска** сообщений в Telegram по ключевым словам с использованием библиотеки **Telethon** (MTProto API).

Позволяет находить сообщения в публичных группах, супергруппах и каналах, доступных через глобальный поиск Telegram, и сохранять результаты в JSON.

## Возможности

- Глобальный поиск по одному или нескольким ключевым словам  
- Ограничение количества результатов на каждое ключевое слово  
- Фильтрация по дате (начиная с указанного дня)  
- Сохранение результатов в JSON-файл  
- Автоматическая обработка `FLOOD_WAIT` и сетевых ошибок  
- Собираемая информация:  
  • текст сообщения  
  • название чата  
  • отправитель  
  • дата отправки  
  • ссылка на сообщение (если доступна)

## Важные ограничения

Метод `messages.searchGlobal` ищет **только** в:

✅ публичных каналах  
✅ публичных группах и супергруппах  
✅ чатах, видимых в глобальном поиске Telegram

**Не ищет**:

✗ в приватных группах, куда ваш аккаунт не добавлен  
✗ в приватных/закрытых каналах без username  
✗ в личных переписках других пользователей  
✗ в удалённых или скрытых сообщениях

Это именно глобальный поиск Telegram, а не полный доступ ко всем чатам платформы.

## Требования

- Python 3.9 или новее  
- Действующий Telegram-аккаунт  
- API-ключи (`api_id` и `api_hash`)

### Работа под Windows
## Установка
```bash
git clone https://github.com/oulabla/tg-keyword-searcher

cd tg-keyword-searcher

python -m venv venv

venv\Scripts\activate

pip install -r requirements.txt

python main.py "bitrix"
```
## Запуск
В папке где скрипт
```bash
venv\Scripts\activate
python main.py "bitrix"
```

## Получение API данных

1. Перейдите на https://my.telegram.org
2. Авторизуйтесь по номеру телефона
3. Выберите **API development tools**
4. Создайте новое приложение (любое название и описание подойдёт)
5. Сохраните полученные значения:

   - `api_id`  
   - `api_hash`

## Первый запуск

При первом запуске скрипт запросит:

- `api_id`
- `api_hash`
- номер телефона (в международном формате, например: `+71612345678`)

После успешной авторизации будет создан файл `cred.yaml`.  
При всех последующих запусках данные будут автоматически браться из этого файла.

## Использование

```bash
python main.py "ключевое слово" -l 100 -o data.json
```

| Флаг              | Описание                                          | По умолчанию    |
|:-----------------:|:--------------------------------------------------|:---------------:|
| `-l, --limit`     | Максимум сообщений на каждое ключевое слово       | `50`            |
| `-o, --output`    | Имя выходного JSON-файла                          | `results.json`  |
| `-j, --json-only` | Выводить только чистый JSON (без логов в консоль) | —               |
| `--since`         | Искать сообщения начиная с даты (YYYY-MM-DD)      | без ограничения |

### Ограничения Telegram

Telegram ограничивает частоту глобального поиска. Возможны ошибки:

FLOOD_WAIT_X

Скрипт автоматически делает паузу и продолжает работу.

Неполные результаты

Глобальный поиск Telegram:

ранжирует выдачу

не возвращает весь архив

может скрывать старые сообщения

Это нормальное поведение API.
